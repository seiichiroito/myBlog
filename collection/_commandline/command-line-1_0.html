Learn Enough Command Line to Be Dangerousとは完全な初心者のためのコマンドライン入門であり、出来るだけ多くの人に”コンピューターマジック”(Box1.1)の共通の基礎を教えるよう設計されたチュートリアルシリーズの一つ目である。

このチュートリアルは、ソフトウェア開発者と働く人やソフトウェア開発者を志す人両方にむけて作られている。比較的高度な技術的素養を想定している他の多くのコマンドラインチュートリアルと違い、*Learn Enough Command Line to Be Dangerous*は一般的なコンピュータの知識( アプリを立ち上げられる、ウェブブラウザを使える、タッチタイプができるなど )以上の前提知識は想定していない。とりわけ、テキストエディタの使い方、あるいはテキストエディタとは何かを知っているかことを想定していない。それどころか、このチュートリアルではコマンドラインとは何かを知っているかも想定していないので、もしタイトルで困っていても、あなたはまだ正しい場所にいる。

最後に、仮にもし、既にコマンドラインの使い方を知っていても、このチュートリアルを読み進めることで（また演習をすることで）、いくらか知識が補完され、またおそらく新しいことをいくらか学ぶことができるだろう。

Box 1.1 コンピューターマジック

コンピュータは、現実世界で私たちが獲得してきた中で最も魔法に近いものかもしれない。機械に呪文を打ち込み、そして、 ー もし呪文が合っていたら ー 機械はその命令を行う。そんな魔法を行うために、コンピュータの魔女と魔法使いは言葉だけでなく、杖、ポーション、そしていくつかの古代の本に頼っている。

一括りにし、これらの技はソフトウェア開発として知られている。コンピューターブログラミングに加え、コマンドラインやテキストエディタ、バージョン管理、またそれらのツールの知識を持つことは、おそらく技術者とそうでない人の主な境界線となるだろう。（もしくは魔法用語で言えば、魔法使いとマグルとの間の違いとなる。）このチュートリアルはこの技術者とそうでない人の境界線を越える最初のステップとして存在している。技術的素養を身につけることで私たちはソフトフェアの魔法使いとなり、コンピューターの呪文を詠唱し、命令を実行してくれる機械を得ることができるだろう。

私はMichael Hartl。本とスクリーンキャストの両方で構成された、ウェブ開発入門の一つであるRuby on Railsチュートリアルの製作者としてよく知られているであろう。(またおそらく、より数学的側面であるTau Dayの創設者、The Tau Manifestoの著者として私を知っている方もいるだろう。)

Railsチュートリアルに関してよくされる質問の一つにこんなものがある。「完全な初心者に Railsチュートリアルは向いているでしょうか?」その答えは、「あまりお勧めできない。」である。Ruby on Railsチュートリアルによって完全な初心者がウェブ開発を学ぶことが可能な一方で(実際多くのことを学ぶだろう)、挑戦的でたまにイライラすることもあり、一般的に勧めていない。代わりに私はこのチュートリアルから始めることを勧めている。

多くのプログラミングのチュートリアルはコマンドラインに関して、ごまかすか既に使い方を知っていることを想定している。しかしコマンドラインの基礎を理解することは技術力のある開発者になるために絶対に不可欠である。

経験豊富なコンピュータプログラマーのデスクトップを見ると、macOSのような精錬されたグラフィカルインターフェイスを搭載したシステム上でさえ、それぞれに一連のコマンドを含む、ターミナルウィンドウがたくさんあるだろう。（図1.1）。コマンドラインを熟練することはプロダクトマネージャーやプロジェクトマネージャー、デザイナーなどの開発者と働く必要のある人にとっても役立つ。この不可欠な技術的素養をできる限り多くの読者に理解してもらうことがLearn Enough Command Line to Be Dangerousのゴールである。
