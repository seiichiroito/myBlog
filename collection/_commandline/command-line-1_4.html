コマンドラインを実行しているプログラムは専門的にshellと呼ばれる。 shellは利用可能なコマンドについて学ぶことができる、強力な（時に不可解な）ツールを含んでいる。

このツールとはそれ自身がコマンドラインのコマンドでもあるman（”manual”の省略形）である。引数としてコマンドの名前を取り（echoのような）、そのコマンドについてより詳しく知ることができる。詳細はシステムによって異なるが、私のシステムでman echo実行した結果がリスト1.3である。

リスト1.3: man echoを実行したときの結果

$ man echo

ECHO(1) BSD General Commands Manual ECHO(1)


NAME


echo -- write arguments to the standard output


SYNOPSIS

echo [-n] [string ...]


DESCRIPTION

The echo utility writes any specified operands, separated by single blank (` ') characters and followed by a newline (`\n') character, to the stan- dard output.


The following option is available:


-n Do not print the trailing newline character. This may also be achieved by appending `\c' to the end of the string, as is done by iBCS2 compatible systems. Note that this option as well as the effect of `\c' are implementation-defined in IEEE Std 1003.1-2001 (``POSIX.1'') as amended by Cor. 1-2002. Applications aiming for maximum portability are strongly encouraged to use printf(1) to suppress the newline character.

:

リスト1.3の最後の行には:があり、まだ下に情報があることを指し示している。この最後の行の詳細はシステムによって異なるが、矢印下キーを押すかスペースキーを押すことで一行ずつ次の情報を見ることができるだろう。 manページを出るには、”q”（”quit”の意）のキーを押す。（このmanページの画面構成は章3.3で学ぶlessプログラムと同じ構成になっている。）

man自身もコマンドのため、manにmanを適用することができる（図1.10）。その結果はリスト1.4の通りである。

リスト1.4: man manを実行したときの結果

$ man man

man(1) man(1)


NAME

man - format and display the on-line manual pages


SYNOPSIS

man [-acdfFhkKtwW] [--path] [-m system] [-p string] [-C config_file] [-M pathlist] [-P pager] [-B browser] [-H htmlpager] [-S section_list] [section] name ...


DESCRIPTION

man formats and displays the on-line manual pages. If you specify sec- tion, man only looks in that section of the manual. name is normally the name of the manual page, which is typically the name of a command, function, or file. However, if name contains a slash (/) then man interprets it as a file specification, so that you can do man ./foo.5 or even man /cd/foo/bar.1.gz.


See below for a description of where man looks for the manual page files.


OPTIONS

-C config_file

:

リスト1.4から、manの概要はこのようになっている。

man [-acdfFhkKtwW] [--path] [-m system] [-p string] ...

これが”時に不可解”、と最初にmanページについて説明したときの意味である。確かに、多くの場合においてmanページの詳細はほとんど理解不能だと私は感じるが、あるコマンドラインの高レベルな概要を得るためにmanページを読めることは有益なスキルであり、取得する価値のあるスキルである。

manページを読むのに慣れるために、新しいコマンドに出会った時にman <command name>と実行することを勧める。仮に詳細が全て明白ではないとしても、manページを読むことは技術的素養の価値あるスキルを養う手助けとなる（Box1.5）。

Box1.5. 技術的素養

数学では、多くの科目が、純粋な推論を少数の仮定や原理に適用することで作られている。例えば代数学、幾何学、数論、解析。結果として、それらの科目は完全に自己完結しており、したがって正式な前提条件ない。

なので原則として、小さな子供でさえ学ぶことができる。だが、はたまた原則として、他の何かを必要としており、数学者はたびたび数学に関する成熟度を非正式的な前提知識として必要であると言っている。この数学に関する成熟度とは、数学の証明を理解したり書いたりすることができるような経験や一般的な素養を培っていることを指している。

テクノロジー界では、似たようなスキル（もしくは、より正確に言えばスキルのセット）が技術的素養という形で存在している。テキストエディタやUnixコマンドラインなどでよく知られている”ハードスキル”に加えて、技術的素養はどこに目的のメニューアイテムがあるか探せたり、Googleを使ってどう調べたいことを検索するかを知っていたりといった”ソフトスキル”を含んでおり（"テックサポートチートシート"としてskedでイラストされている）、どのように機械に命令をさせるかという姿勢とともに存在している（Box1.1）。

それらのソフトスキル、そして姿勢は直接教えるのが難しいため、このチュートリアルとその後に続くLearn Enoughチュートリアルをやっていく中で技術的素養を鍛える機会を常に探し続ける必要がある。（例えばmanページを使うことでプログラムの要点を知る方法を学ぶなど。(章1.4)）やがて、その探究量が累積し、”テックサポートチートシート”の著者のような、全てのプログラムにおいてなんでもできるような、外見上魔法のような能力を得ることができるだろう。

ところで、”テックサポートチートシート”は一般的な問題を解決するのに大切な三つの技術が抜けている。

1. アプリケーションをリスタートしたか。

2. デバイスを再起動したか。

3. アプリをアンインストールしてインストールしなおしたか。

これらのステップは通常試すべき順序を示しており、二つめだけでおそらく90%の予期せぬコンピュータのエラーを解決することができるだろう。

1.4.1 演習

1. Manページを参考にし、あなたのシステムにおける`echo`の正式な説明について、短い説明と長い説明それぞれ調べなさい。

2. リスト1.2で見た通り、標準のechoコマンドは引数をスクリーンに出力し、新しいプロンプトを新しい行に追加する。これは”ニューライン(newline)（文字通り新しい行に文字を追加する。多くの状況で”バックスラッシュ n”\nと書かれる。）”と呼ばれる特殊な文字によるものである。`echo`は、新しい行によって分けられない一連の文字列を出力するためのプログラムでたびたび使われるので、ニューラインによって新しい行が挿入されるのを防ぐ特別なコマンドラインオプションがある。

echoのmanページを読むことで新しい行に行かずに”hello”と出力するコマンドを突き止め、予想通りに動くかターミナルで確かめなさい。

ヒント: コマンドラインのオプションの場所を突き止めるには、図1.5を参照するといいかもしれない。リスト1.5、リスト1.6と実行した結果を見比べ、正しくオプションを使えたことを確認できるだろう。 (Note: この演習はmacOSの古いバージョンで標準ターミナルを使っていると失敗するかもしれない。この場合、iTermをインストール（とにかく悪い選択肢ではない。)することをお勧めする。)

リスト1.5: echoをニューライン（オプションなし）と一緒に実行した結果

hello

[~]$

リスト1.6: echoをニューライン（オプションあり）なしで実行した結果

hello[~]$
