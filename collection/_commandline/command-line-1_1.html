「最初は... コマンドラインだった。」

グラフィカルユーザインターフェース(GUI)はコンピュータの使い方を劇的に簡単にしうるが、多くの場合に置いて一番強力で柔軟にコンピュータと対話できるのはコマンドラインインターフェース(CLI)である。そのようなインターフェースで、ユーザーはコマンドを打つことで望んだ仕事をコンピュータに指示することができる。コマンドは様々な結果を達成するために様々な形で組み合わせられる。典型的なコマンドラインのコマンドの例が図1.2である。

このチュートリアルはUnixコマンドラインの基礎をカバーしている。UnixとはLinux, Android, iOS（iPhoneとiPad)、そしてmacOSを含んだオペレーションシステムの一種を指している。UnixシステムはほとんどのWorld Wide Web上のソフトウェアに使われ、ほとんどの携帯、タブレットを動かし、世界の多くのデスクトップコンピュータにも搭載されている。近代のコンピューティングにおいてUnixが中心的な役割を担っているため、このチュートリアルはUnixによるソフトウェア開発をカバーしている。

Unix黄金時代における主な例外としてMicrosoft Windowsがあり、WindowsはUnixを使っていない。しかし、ほとんどWindowsの開発ツールを使って開発を行う人にとってもUnixコマンドラインを学ぶことによって得られる恩恵があるだろう。とりわけ、ある時点でそのようなデベロッパーはUnixサーバーでコマンドラインを使うことがあり（例えば”セキュアシェル”コマンドとしてsshを介して)、そういった状況ではUnixコマンドに慣れていることは不可欠である。そのため、Windowsユーザーは仮想マシンをインストールするか（Box1.2）、Enough Dev Environment to Be DangerousのWindowsユーザー向けの手順に従ってLinux互換の開発環境をセットアップすることをお勧めする。

もう一つの選択肢として、コマンドラインを内蔵している、Cloud IDEを使うことである。この選択肢も*Learn Enough Dev Environment to Be Dangerous*にてカバーされている。

Box 1.2. 仮想マシンを動かす

Windowsユーザーの一つの選択肢はWindowsでLinuxのオペレーションシステムを使えるようになる仮想マシン(VM)を動かす無料のプログラムをインストールすることである。Windowsはここ数年、Linuxのサポートを改善させてきたので、仮想マシンを動かすために、Learn Enough Dev Environment to Be DangerousのWindowsユーザー向けの手順をまず初めに試すことをお勧めする。

Note：この手順はかなり手際を要する可能性があり、またあいにく仮想マシン上のそれぞれの技術的なサポートを提供するような資料は持っていない。もし何か問題が起こったら、代わりにCloud IDEを使うことをお勧めする。

仮想マシンをインストールする手順は以下のとおりである。

1. pcのシステムにあった正しいVirtualBoxをインストールする(無料)

2. 「Learn Enough Virtual Machine」をダウンロードする

3. ダウンロードが完了したら、”OVA”ファイルをダブルクリックし、仮想マシン（VS）を表示されている手順に沿い、インストールする。

4. 仮想マシンをダブルクリックし、デフォルトのユーザーパスワード、”foobar!”でログインする。

(これらのステップを全て済ませるのは、章1.4で開発し始めてから使う技術的素養を鍛える良い演習である。)手順を終えると図1.3のように、このチュートリアルのために事前に設定された、Linuxのデスクトップ環境が表示される（コマンドラインのターミナルを含む）。

長い目で見て、pcをMacにできるだけ早く変えることをお勧めする。Macは一般的にWindowsより高いので、少し節約しなければならないかもしれないが、多くの場合において、生産性が向上することによってその値段の違いは埋め合わせられるだろう。（もしLinuxが好きでそれを使い続けたいと感じたとしても、Macは一般的にユーザーインターフェイスが優れており、扱いやすく、またMac上でも仮想マシン内でいつでもLinuxを動かすことができる。)
