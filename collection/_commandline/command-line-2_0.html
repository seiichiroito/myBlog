基本的なコマンドの実行方法は学んだので、コマンドラインで最も大事なタスクの一つであるファイル操作を学ぶ準備ができた。 このチュートリアルは技術的な前提知識を想定していないので、テキストを編集するために作られたプログラムの使い方を知っている必要はない。 （テキスト•エディタと呼ばれるそのプログラムは、次のチュートリアルであるLearn Enough Text Editor to Be Dangerousの主題である。）

これはつまりコマンドラインにて手動でファイルを作らならければならないことを意味する。ちなみにこれは仕様であり、バグではない。（Box2.1） コマンドラインでファイルを作る方法を学ぶことはそれ自身で貴重なスキルである。

Box2.1. "オタク語"を話すことを学ぶ

ソフトウェア開発を学ぶ上で大事なことの一つは、ハッカーやマニア、オタクの文化に精通することである。例えば、"それはバグではなく、仕様である。"というフレーズは、欠点を美点として扱うよくある手法を意味する。 Urban Dictionaryではこう定義している。

それはバグではなく、仕様である。

プログラムの中の欠陥を実際そのような動きをするものであるとユーザに納得させようとするときにソフトウェア開発者が言い訳として使う言葉。

ハッカー界隈の膨大で面白い語彙集を含んでいるJargon（専門語）ファイルでは、仕様というタイトルの記事でこのテーマを展開している。

"ドキュメントに書かれていない仕様"というのはバグのよくあるユーモアな言い回しのようである。 ここに"一問オタクテスト"と呼ばれることもある関連したジョークがある。

あなたは誰かにこう言う。

「今日、仕様(FEATURE)と書かれたうぬぼれたプレートをつけたフォルクスワーゲン•ビートルを見たんだ。」

もし彼/彼女が笑ったら、彼/彼女はオタクである。

このジョークは"バグ"が"フォルクスワーゲン•ビートル”を表す一般的なスラングであるところからきている。 仕様と書かれたうぬぼれたプレートをつけているビートルは、さながら"バグではなく、仕様である"が現実に現れたものと言えるだろう。(図2.1)

もしあなたがオタクでもマニアでもなくても、"オタク語を話す"ことを学ぶことは、技術的展望とそれを取り巻く世界の両方を渡り歩く手助けとなるだろう。
