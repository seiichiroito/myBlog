コマンドラインのコマンドを実行するのに、まず初めにコマンドラインを使えるターミナルを始めなければならない。正確な詳細については使っているオペレーションシステムによって異なる。

macOS

macOSでは、⌘␣（コマンド-スペース）か画面の右上にある虫眼鏡のマークをクリックすることで開けるSpotlightを使うことでターミナルのウィンドウを開くことができる。一旦Spotlightを立ち上げたら、Spotlightの検索バーに”terminal”と打ち込むことでターミナルを始めることができる。(もしより高度で、カスタマイズ可能なターミナルに興味があったらiTermを使うことをお勧めするが、任意である。)

この時、リスト1.1のような警告が現れるかもしれない。

リスト1.1: マック上でのターミナルの警告

The default interactive shell is now zsh.

To update your account to use zsh, please run 'chsh -s /bin/zsh'.

For more details, please visit https://support.apple.com/kb/HT208050.

[~]$

この警告はmacOS Catalinaで変更された結果によるものである。これに対して現状何かをする必要はない。この警告に関連しては、このチュートリアルにおいて違いが出るときに改めて言及する。詳細については、Learn Enough内のブログ、”Using Z Shell on Macs with the Learn Enough Tutorials”を参照。

Linux

Linuxでは、図1.3で表示されているようにターミナルアイコンをクリックすることでターミナルを始められる。正確な詳細はシステムによって異なる可能性があるが、結果として図1.4のようなものが表示されるはずである。

Windows

Windowsでは、Linux（驚くべきことに、MicrosoftはネイティブにLinuxをサポートすることを決めた）をインストールをお勧めする。詳しくは無料の*Learn Enough Dev Environment to Be Dangerous*のWindowsの節にて説明してある。一旦Linuxをインストールしたら、上のLinuxで説明してあるとおり、ターミナルのアイコンが表示されるはずである。もし行き詰まったら、技術的素養を駆使して解決するように。

Terminal window

どのオペレーションシステムを使っているにかかわらず、詳細は異なるかもしれないが、ターミナルウィンドウが図1.4のように表示されるはずである。

図1.2で見た例はコマンドラインにおける典型的な要素の全てを含んでおり、[図1.5](#figure-1_5)のようになっている。プロンプト（ユーザーに何かをさせるよう、”鼓舞(prompt)”する）、そのあと続いて、コマンド（”コンピューターに命令(command)を与える”の意）、オプション（”異なる選択肢(option)から選ぶ”の意）、そして引数（数学の”関数の引数(argument)”と同様）となる。プロンプトはターミナルによって自動的に出力されることを理解することが不可欠であり、打ち込むことはない。（実際に打ち込んだとしても、エラーが起こるだけだろう。）そのうえ、実際のプロンプトの詳細は異なることがあるが、このチュートリアルにおいてその違いが重要になることはない（Box1.3）。

{% include component/figure.html num=5 %}

Box1.3. プロンプトとは？

コマンドラインはユーザーにアクションを起こしてもらうために、毎回いくつかのシンボルか”鼓舞(prompt)”させるデザインのシンボルで始まる。プロンプトは一般的にドルサイン`$`か`%`、そしてその前にシステムによって変化する情報が付く。

例えば、あるシステムではこのプロントはこのような表示になる。

Michael's MacBook Air:~ mhartl$

図1.では、プロンプトは代わりにこのような表示だった。

[~]$

また、図1.5ではこのようだった。

[project]$

最後に、私が今見ているプロンプトはこのような表示である。

[learn_enough_command_line (master)]$

このチュートリアルの目的としては、プロンプトの詳細は大事ではないが、次のチュートリアル(*Learn Enough Text Editor to Be Dangerous*)でプロンプトのカスタマイズの便利な方法について記載している。

1.2.1 演習

Learn Enough Command Line to Be Dangerousでは多くの演習を含んでいる。次の章に進む前に演習に取り組む習慣をつけることを強くお勧めしている。それらはその章でやった内容を補完し、言及された多くのコマンドを使うことで、不可欠な実践を行うことができる。

概して、演習を済ませる必要はないので、もし詰まってしまったらそのまま先に進んであとでまたやり直すのもいいだろう。それにまた、本筋のためにも演習をするのは良い助けとなる。見たところ不可能なアイデアや手に負えないような問題でも２回目以降には簡単に見えることがしばしばあることに驚くだろう。

演習の解決方法はLearn Enoughでチュートリアルを購入することでlearnenough.com/solutionsにて見ることができる。他の人の答えを見て、自分の回答を記録するためには、learnenough.com/societyでLearn Enough Societyに参加する必要がある。

1. 図1.5で言及したように、図1.6において、プロンプト、コマンド、オプション、引数、そしてカーソルがどれかを答えなさい。

2. 多くの現代のターミナルプログラムは複数のタブを作ることができる機能がついている(図1.7)。これは関連する一連のターミナルウィンドウを管理するのに役立つ。ターミナルプログラムのメニューバー(図1.8)を調べることで、どうやったら新しいタブを作ることかできるか見つけなさい。

追加事項：新しいタブを作るキーボードショットがどれかも見つけること。（システムのキーボードショートカットを学ぶことは、技術的素養を磨くのに素晴らしい習慣である。）
